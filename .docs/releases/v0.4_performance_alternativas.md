# Release v0.4 - Refatoração de Performance + Alternativas Estruturadas

## Objetivo
Eliminar o lag de 1s nas interações e evoluir o modelo de dados das questões para suportar alternativas e explicações estruturadas.

## Mudanças

### Performance
- `Groq(...)` agora é instanciado via `@st.cache_resource` — conexão HTTP criada 1 única vez e mantida em memória entre reruns do Streamlit.
- TTL do cache de leitura dos CSVs aumentado de 10s para 30s.
- Cache invalidado explicitamente (`st.cache_data.clear()`) sempre que um CSV é escrito, garantindo dados frescos sem polling desnecessário.

### Schema do CSV (dados_questoes.csv)
Novas colunas adicionadas:
- `Alt_A`, `Alt_B`, `Alt_C`, `Alt_D` — texto completo de cada alternativa
- `Explicacao` — justificativa da resposta correta gerada pela IA ou preenchida manualmente

Migração suave: a função `_garantir_arquivos()` detecta e preenche as colunas faltantes em CSVs antigos.

### IA (Groq / Llama 3.3)
- Prompt atualizado para retornar JSON com 7 chaves: `pergunta`, `alternativa_a/b/c/d`, `resposta_correta`, `explicacao`.
- Parsing atualizado para mapear cada chave em sua respectiva coluna do CSV.

### UI — Responder Questão
- Radio buttons exibem o **texto completo** de cada alternativa: `A) Usar Azure Data Factory...`
- Barra de progresso mostra `X/Y questões respondidas` no topo da fila.
- Após responder, um expander `💡 Ver Explicação` aparece com a justificativa, quando disponível.

### UI — Cadastro Manual
- Formulário agora inclui 4 campos de texto para as alternativas A, B, C, D.
- Campo opcional de `Explicação`.

### Correções
- Deprecation `use_container_width` corrigido para `width='stretch'` nos gráficos Plotly.
- Cálculo de acertos no Dashboard agora é case-insensitive.

## Status
- **Finalizado**: App subiu limpo, sem warnings de deprecação.
