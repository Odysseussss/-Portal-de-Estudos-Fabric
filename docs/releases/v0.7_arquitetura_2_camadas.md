# Release v0.7 — Arquitetura de 2 Camadas (Público vs Admin)

## Objetivo
Separar o portal em dois modos de execução para viabilizar o deploy público como portfólio, sem expor funcionalidades administrativas nem chaves de API.

## Mudança de Arquitetura

### Detecção Automática de Modo
- **Modo Admin** (sua máquina): Detectado pela presença de `GROQ_API_KEY` no `.env`. Exibe menu completo com geração de IA e configurações.
- **Modo Público** (produção): Sem `.env` → sem chave → as páginas de admin desaparecem completamente do menu. Nenhum código de IA é sequer importado.

### Respostas por Sessão (`st.session_state`)
- As respostas dos usuários agora vivem em `st.session_state.respostas_usuario`, isoladas por sessão do navegador.
- O CSV `dados_questoes.csv` é tratado como **conteúdo editorial read-only** — as questões pré-geradas viajam junto no repositório.
- Cada aluno tem seu próprio progresso, acertos e erros, sem interferir nos dados de outros.

### Kanban por Sessão
- O sistema de tarefas foi migrado de CSV para `st.session_state.kanban_tarefas`.
- Cada usuário cria e gerencia suas próprias metas de estudo.
- Adicionado botão de deletar tarefa (🗑️) nos cards do Kanban.
- Os dados não persistem entre sessões — são pessoais e temporários por design.

### Segurança
- Removido o sistema de login/senha (desnecessário — se não há chave, não há admin).
- A importação de `groq` e `dotenv` é condicional: em produção, nem precisam estar instalados.
- Nenhuma rota administrativa é renderizada sem `GROQ_API_KEY`.

### Menu Dinâmico
| Modo | Menu |
|------|------|
| Público | Dashboard, Praticar Questões, Minhas Metas |
| Admin | Dashboard, Praticar Questões, Minhas Metas, **Gerar Questões (IA)**, **Configurações** |

## Fluxo de Trabalho
```
Você (local) → Gera questões com IA → commit do CSV → push → Deploy atualiza
Aluno (web) → Responde questões do banco → Vê dashboard → Cria metas no Kanban
```
